-----COMANDOS-----

'git init' => inicializa o git na pasta;

'git status' => é utilizado para ver o que tem de alteração(arquivos novos, deletados, editados) na pasta;

'git add "nome_do_arquivo"' => para adicionar o arquivo (nome do arquivo entre aspas duplas);
'git add .' => para adicionar todos os arquivos mencionados;

'git commit -m  "texto explicativo do commit"' => utilizado para fazer o commit das alterações;

'git config --global user.email "natan2624@gmail.com"' => comando para configurar o email da conta do git(somente na primeira vez caso não tenha sido feito);
'git config --global user.name "Natan O Pelizzoni"' => comando para configurar o nome da conta do git(somente na primeira vez caso não tenha sido feito);

'git remote add origin https://github.com/NatanOPelizzoni/Curso-Git' => setando o repositório criado no github;

'git push --set-upstream origin master' => faz o push do commit para a branch master;

'git reflog' => mostra um log com todas as alteracões feitas, mostrando a mais recente no topo da lista;

'git reset --hard idDoCommit' => utilizado para restaurar algum dos commit existentes(idDoCommit = cód que aparece no inicio do reflog) Ex.: 'git reset --hard f2fdadf'. !!!Realizar esse comando somente se necessário!!!;

'git branch' => utilizado para mostrar as branchs disponiveis no momento;

'git branch nomeDaBranch' => comando para criar uma nova branch;

'git checkout nomeDaBranch' => comando para trocar para a branch que deseja trabalhar;

'git checkout -b nome-da-nova-branch nomeDaBranchOrigem' => cria uma nova branch apartir de uma existente(de preferencia da mais atualizada)


-----OBSERVAÇÕES-----
realizar a seguinte sequencia de comandos para subir as alterações feitas:

-> 'git status' para verificar as alterações que foram feitas nos arquivos
-> 'git add .' para adicionar ao commit todos os arquivos novos, editados e apagados
-> 'git commit -m "mensagem do que foi feito"' para fazer o commit das alteracões

-> 'git push' para subir uma nova versão "final"


para fazer merge entre branchs

-> 'git checkout nomeDaBranchDestino' => para entrar na branch que vai receber as alterações

-> 'git pull' => para baixar a versão final da branch principal, para que não de problema de perder código

-> 'git merge nomeDaBranchOrigem' => para fazer o merge da nomeDaBranchOrigem para a nomeDaBranchDestino

-> 'git push' => para subir a versão final da branch


criar uma nova branch

-> 'git checkout master' => para entrar na master

-> 'git pull' => para pegar a versão mais atualizada

-> 'git checkout -b nova-branch-teste master' => para criar a nova branch a partir da master


ignorando arquivos, pastas, ... (não adicionando nas branchs)

-> 'touch .gitignore' => necessário ser pelo terminal do git

dentro do arquivo .gitignore adicionar da seguinte foram

nomeDaPasta/ => para ignorar pastas
nomeDoArquivo.extensão => para ignorar algum arquivo específico
